Review peer R package

Last updated on Oct 26, 2020 4 min read

Overview

This post is going to review my peer Nguyễn Đăng Thành R package icecovid19. Nguyễn Đăng Thành and I are both in Master of Business Analytics program in Monash University and this package was an assessment of unit Communicating with Data. The package is aiming to launch the shinyapp made before and refactor some output as function. Great job! Thanh!

To download the package, you can

# install.packages("devtools")
devtools::install_github("etc5523-2020/r-package-assessment-Thanh-8213")

The data set in package:

  • data_total

  • data_total_newdaily

The functions in package:

  • launch_app

  • addLabel

  • getSummaryDT

  • sliderDateinput

  • summariseData

  • summariseData_newdaily

What did I think this package would do? Did it do it?

Following the instruction of README.md, I downloaded icecovid19 package successfully in my rstudio. The package passed R-CMD and pkgdown checks. Thanh inserted data_total and data_total_newdaily two data set in package and when typed in the console, the data set structure and variable will display. The function getSummaryDT is a refactor DT table in the shinyapp as the function output. Function launch_app can load Thanh’s shinydashboard successfully. The function which refactor the shinyapp in ui and server were passed the test.

Data

  • data_total

The dataset contains the number of COVID19 cases worldwide from 22/01/2020 to 06/10/2020 as well as the date and the region in which cases were recored. There are 3 type of cases: Confirmed, deceased and recovered

  • data_total_newdaily

The dataset contains the number of COVID19 cases worldwide from 22/01/2020 to 06/10/2020. Data is in longer format, with variable vars contains 6 types of cases: Confirmed, deceased, recovered, new daily confirmed, new daily death and new daily recovered.

Functionality

  • launch_app This function designed for launch the shinyapp.

  • addLabel This function create HTML label for the types of cases. When mouse is hovered on leaflet map, the label will appear and give addition details.

  • getSummaryDT This function create a summary data table that shows the detailed number of confirmed, deceased and Estimated recovered cases.

  • sliderDateinput The function constructs a slider widget to select a date value from a range of value. Thanh test this ui refactor function and congratulate the test pass!

  • summariseData The function summarise data creates a new data frame. It will have one row for each combination of grouping variables.

  • summariseData_newdaily Summarise Data newdaily daily new cases

Improvement suggestion

The shiny dashboard that Thanh made is extraordinary that I really like the click event function inset the leaflet map. If I had to make suggestion I would suggest that adjust the plotly position in the plots section and look align would be better. It is better to add more introduction in README.md and regretted that Thanh set his pkgdown website as invisible that the website of the package was missing that it not good for overview the package.
